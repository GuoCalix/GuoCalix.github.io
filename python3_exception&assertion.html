Exception and Assertion

In this note we will introduce some useful techniques to handle exception and assertion in python3.

Exception

Different from a syntax error or a parsing(语法) error, an exception is an error detected while executing the program.(That's why an exception is also called a runtime error.)
There are many types of exceptions, some are built-in-type and others are exceptions built by programmers.
In the following parts we will cover some common types of exceptions, and some statements useful in handling exceptions.

Compound Statement

Compound statements contains group of other statements. The if, while and for statements implement traditional control flow constructs. try statement specifies exception handlers (like except, try) and cleanup code (like finally can ensure the execution of the releasing(cleanup) code). with statement allows the execution of initialization(初始化) and finalization(复原化) around a block of code. Function and class definitions are also syntactically compound statements.

Common Built-in Exception

  • NameError: Raised when a local or global name is not found. This applied only to unqualified name [1].
  • UnboundLocalError: This is a subclass of NameError, raised when a reference is made to local variable in a function or method, but no value has been bound to that variable.
  • IndexError: Raised when a sequence subscript is out of range.
  • TypeError: Raised when an operation or function is applied to an object of inappropriate type.
  • ValueError: Raised when an operation or function recieves an argument that has the right type but inappropriate value.
  • KeyError: Raised when a mapping(dictionary) key is not found in the set of existing keys.
  • ZeroDivisionError: Raised when divided by zero.
  • AssertionError: Raised when an assert statement failed.(see more in the assertion part)
  • AttributeError: Raised when an attribute reference or assignment fails. (if an object do not support attribute, a TypeError is raised)[2]

try and except

We can use try and except to handle exception.

  • execute the try clause
    • if no exception, skip the except clause
    • if have exception, jump to the except part, compare the exception type with the type after except
      • if type matches execute the except part and continue.
      • if don't match break to the outer try, if no outer try stop with an error message.
while True:
    try:
        x = int(input("please input an integer"))
        y = 1 / x
    except ValueError:
        print("No valid number, try again")
    except ZeroDivisionError:
        print("ZeroDivision Error")

raise exception

The raise statement allows the programmer to force a specified exception to occur.

The sole argument after raise must be either an exception instance or an exception class (a class derives form BaseException, such as Exception or one of its subcalss)

Finally and Else

A finally clause is always executed before leaving the try statement, whether an exception or some extreme condition(like break) is occured or not.

finally:
    print("HelloWorld")

Self-defined Exception and Derived Relation

All built-in, non-system-exiting exception are derived from the Exception class, and user-defined exception must be derived from any of the built-in exception class and is encouraged to derived form the Exception class.
And except clause matches exceptions which are instances of the class or one of its derived class. As an example showed below.

class AError(Exception):
    pass
class BError(AError):
    pass
class CError(BError):
    pass
try:
    raise CError
except AError:
    print("AError is raised")
try:
    raise ZeroDivisionError
except Exception:
    print("An exception is raised")
a = AError()
print(type(a))
print(type(AError))
print(type(RuntimeError))
raise a

with statement

The with statement is used to wrap common try-except-finally usage pattern for convenient use.

f = open('Text_file.txt','w')
try:
    f.write('Hello World!')
finally:
    f.close()

The code can be simplified as:

with open('Text_file.txt','w') as f:
    f.write('Hello World!')

Assert statement

The assert is not a kind of compound statements while it's a kind of simple satatements.
The assert key word let you test if a condition in your code returns True, if not, the program raise an AssertionError.
usage:
assert expression1 [, expression2] is equivalent to

if __debug__:
    if not expression1 : raise AssertionError(expression2)

for example:

def test_assert(r):
    assert 0 < r < 1 , \
    f"{r=} is expected to be between 0 and 1"
    print(f"{r=} is valid")

test_assert(0.2)
test_assert(0.7)
test_assert(1.2)
