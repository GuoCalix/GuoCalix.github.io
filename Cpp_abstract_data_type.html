Programming with abstraction

Abstract Data Type

  • Domain: a set of values with abstract representation.
  • Operation: the Only Way to access abstract values.
    • No way to directly inspect the physical value of ADT.
  • Classes : User defined ADTs in C++.

String

For example, string is an internal ADT, defined in C++ standard library <string>.

  • getline method (C++11)

      (1)	
  istream& getline (istream&  is, string& str, char delim);istream& getline (istream&& is, string& str, char delim);
  (2)	
  istream& getline (istream&  is, string& str);istream& getline (istream&& is, string& str);

    Extract characters from is and stores them into str until the delimitation character delim or newline character '\n', for(2). The delimiter is discarded.

  • Method calls
    <variable>.<mathod>(<arguments>)

    • str.length()
    • operators are methods.
    • concatenation: str1 += c; str1 + str2
    • compare: lexicographic order.
    • substr: str.substr(pos,n); str.substr(pos);
      • create a new string by extracting n characters starting from pos.
    • find: find the first character or string from pos
      str.find(chr ,pos = 0); str.find(str1 , pos = 0)
    • Delete n characters from str starting at pos:str.erase(pos,n);
    • Insert a copy of str1 into str starting at pos:str.insert(pos, n);
    • Replace the n characters in str starting at pos with a copy of str1: str.replace(pos, n, str1);

Reference

A Cpp reference is a simply reference data type that is less powerful but safer than the pointer type inherited from C. And a Cpp reference is what an alia is called in Cpp.
When mutiple names are refered to the same store(object) in the memory, we call them alias.

  • Call by name:

    <type> <name>(…, <type>& id, …) {
the function body may read or write id
}

    // the function is not able to change the id in the body.
<type> <name>(…, const <type>& id, …) {
 the function body may only read id
}
    • No copying of values happen.
    • The mapping from names to stores is called a symbol table or an environment.

  • Aliasing of type:typedef <type> name rebind type to a name.

  • Do not use using namespace std in the interface(sth.h)!

Polymorphism (多态性)

Polymorphism: a single interface for different kinds of entities.

  • Ad-hoc polymorphism (Overloading)
    • behave differently depending on parameters.
  • Parametric polymorphism
    • behave uniformly with different types.
  • Subtyping (Class inheritence)
    • subtypes of an ADT have the same interface but may behave differnently.

Overloading

  • Overloaded operators
  • Overloaded methods
  • Overloaded functions
    • behave differently depends on the type of parameters.
    • behave differently depends on the number of parameters.

Parametric polymorphism

    cout << id (65) << endl;

Collections

Collections: Data structure for representing a collection of values.
Collections may be represented as polymorphic ADTs in C++.

Vector

Vector: A finite sequence of values of the same type.
syntax:

vector<T> vec;

initialize and access:

vector<string> cars = {"Volve", "BMW", "Ford", "Mazda"};
vector<string> cars(10,"Volve");//ten Volve
cout << cars[0];
cout << cars.at(0);// .at() is often preferred over [] because it lets you know if an error occurs.

get the first or last elements:

cout << cars.front();
cout << cars.back();

rewrite an element:

cars[0] = "Tesla";
cars.at(0) = "Tesla";

Add and remove elements:

cars.push_back("VW");
cars.pop_back(); //remove an element form the end of the vectors.

vector size and empty check:

cout << cars.size();
cout << cars.empty();

loop through a vector:

for (int i=0; i < vector.size(); i++) {
  cout << cars[i] << endl;
}

for (string car : cars){
  cout << car << endl;
}

insert and erase:

vec.insert(vec.begin()+2,15); //insert 15 to the index 2
vec.erase(vec.begin()+2); // delete the index 2 element

Stack

A stack stores a sequence of values and must be added and removed from the top, and no way to look into the element below the top one.

stack<T> s;

basic methods:

s.push(value); //push value on the top of the stack
s.top(); //return the topmost element
s.pop(); //remove the topmost element

Queue

A queue is literally similar to the stack but it is FIFO (first in first out).
