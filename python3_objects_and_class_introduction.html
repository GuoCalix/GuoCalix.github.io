Data Abstraction

Like before, we have constructed a pair abstraction, now we are going to introduce some more inportant data abstraction in python.

The data abstraction is not a particular data type, but instead a collection of behaviors that different specific types share.

HOF and Data Structure

  • An interesting example: in fact we can make a compound data structure like tuple with function, though it's not python's way for efficiency reason.
    >>> def make_pair(x, y):
      """Return a function that behaves like a pair."""
      def dispatch(m):
        if m == 0:
            return x
        elif m == 1:
            return y
      return dispatch
>>> def getitem_pair(p, i):
      """Return the element at index i of pair p."""
      return p(i)
>>> p = make_pair(1, 2)
>>> getitem_pair(p, 0)
1
>>> getitem_pair(p, 1)
2

Sequence

Shared characteristics:

  • Length:A sequence has a finite length.
  • Element selection:Every elements in the sequence is corresponding to an number in the range $\lbrace a\in N|a<Length \rbrace$. And any number in the range is corresponding to an element.

There are several ways to form a sequence

  • recursive list: contains of the first element and the rest of the list recursively.
    e.g.(1, (2, (3, (4, None))))
    • Find length: Peel the list iteratively until the end mark and count the length.
    • select element: Peel the list till meet the element of the signated number.
  • tuple: A full sequence which can use operators like '[]', '+', and '*'.
  • map: A built-in function relies on the sequence abstraction. Orderly access each elements in a sequence and operate with specified function than return in a sequence.
    >>> alternate = (1,-2,3,-4,5)
>>> new = tuple(map(abs,alternate))
>>> new
(1, 2, 3, 4, 5)
>>> type(map(abs,alternate))
<class 'map'>
>>> new = list(map(abs,alternate))
>>> new
[1, 2, 3, 4, 5]

Features of the object

Mutable and Immutable Objects

There are two types of objects in python: mutable and immutable.Whenever an object is instantiated, it is assigned a unique unchangeable object id. But the state of mutable objects can be changed while the immutable objects can't.

  • Immutable objects in python:int, float, bool, string, tuple, Unicode.
  • Mutable objects in python:list, dic and set.

Difference: Immutable objects is easy to access but expensive to change because it involves the creation of a copy.

Nonelocal and Global

Nonelocal statement mandates that whenever we changed the binding of the name 'xxx', the binding is changed in the first frame in which 'xxx' is already bound.
And without a nonlocal statement, an assignment statement would always bound the first frame of the environment.

>>> def make_withdraw(balance):
...             """Return a withdraw function that draws down balance with each call."""
...             def withdraw(amount):
...                         nonlocal balance
...                         if amount>balance:
...                                     return 'Insufficient funds'
... 
>>> def make_withdraw(balance):
...             """Return a withdraw function that draws down balance with each call."""
...             def withdraw(amount):
...                         nonlocal balance
...                         if amount>balance:
...                                     return 'Insufficient funds'
...                         balance = balance - amount                                                                                                                                           
...                         return balance                                                                                                                                                       
...             return withdraw                                                                                                                                                                  
... 
>>> withdraw = make_withdraw(100)
>>> withdraw(10)
90
>>> withdraw(27)
63
>>> withdraw(60)
3
>>> withdraw(10)
'Insufficient funds'
>>> make_withdraw(100)
<function make_withdraw.<locals>.withdraw at 0x000002CC77F12020>

Why this could happen is detailed in the note python3_mechanism&frame.
When we call make_withdraw again, it will create another frame seperated with the former with_draw and thus the former balance.

>>> withdraw_2=make_withdraw(100)
>>> withdraw_2(10)
90

In a word, the case is that only function call can introduce new frames. Assignment statement only changes bindings in the exsiting frames.

Global statement can be used to indicate that particular variables live in the global frame(scope) and should be rebound there.

def scope_test():
    def do_local():
        spam = "local spam"

    def do_nonlocal():
        nonlocal spam
        spam = "nonlocal spam"

    def do_global():
        global spam
        spam = "global spam"

    spam = "test spam"
    do_local()
    print("After local assignment:", spam)
    do_nonlocal()
    print("After nonlocal assignment:", spam)
    do_global()
    print("After global assignment:", spam)

scope_test()
print("In global scope:", spam)
